# Вечерний синк v2

Вечерний синк работает в контексте команды и её `timezone`.

## Жизненный цикл

1. Менеджер открывает синк через API или Telegram-команду.
2. Создаётся `daily_sync_sessions` со статусом `open`.
3. Сообщения команды в привязанном Telegram-чате проходят semantic routing.
4. Сообщения класса `daily_report` сохраняются в `daily_sync_reports`.
5. При закрытии синка отсутствующие пользователи с активным `absence_periods` не считаются должниками.

## Absence

Сообщения класса `absence_notice` создают `absence_periods`. В этот период напоминания и missing sync не отправляются пользователю, но его задачи остаются в рисках команды.

## Report matcher

Базовое сопоставление использует:

- пересечение токенов с названием задачи;
- совпадение ответственного;
- слова статуса: `готово`, `сделал`, `в процессе`, `жду`, `заблокировано`.

Высокая уверенность обновляет статус задачи. Низкая уверенность оставляет отчёт как принятый и требует уточнения в чате.
