# Дизайн: Геймификация + командный тамагочи

> Статус: **дизайн** (Bucket B). Кода пока нет.
> Идея пользователя: один питомец на всю команду; если настроение сотрудников
> отдела грустное — питомец тоже грустит. Плюс — развить геймификацию в целом.

## Что уже есть

- XP/уровни/ачивки: `user_xp_events`, `user_xp_totals`,
  `team_gamification.py` (`grant_team_xp`, `level_for_points`, `ACHIEVEMENTS`,
  `team_leaderboard_payload`).
- XP начисляется за закрытие задач, саммари, участие во встречах и т.п.
- Лидерборд команды в чате (`/leaderboard`) и в mini-app (`tgapp`).

Тамагочи и расширенная геймификация **надстраиваются** над этим, не заменяя.

## Командный питомец (Tamagotchi)

```python
class TeamPetModel:
    id, team_id (unique)
    name, species
    mood       Float   # 0..1 — настроение
    energy     Float   # 0..1 — «сытость», падает со временем (decay)
    level      Int     # растёт от командного XP
    last_fed_at, last_mood_at, created_at
```

### Настроение питомца = f(эмоции отдела, здоровье задач)

```
mood = w1 * team_emotion_valence        # из EmotionPortrait (см. emotional-portrait.md),
                                         #   если включён opt-in; иначе нейтрально
     + w2 * task_health                  # доля задач без просрочки, скорость закрытия
     - w3 * overdue_pressure             # просроченные/заблокированные задачи
     + w4 * recent_activity              # синки, закрытия за период
```

- Если эмоциональный анализ **выключен** для отдела — mood считается только по
  объективным сигналам задач (task_health/overdue/activity). Питомец всё равно
  работает, просто без «эмоциональной» компоненты.
- `energy` со временем падает (decay); «кормление» = командные действия
  (закрытие задач, вечерние синки, участие во встречах) — те же события, что
  уже дают XP, переиспользуем хук `grant_team_xp`.

### Состояния (для UI)

`happy` / `content` / `neutral` / `tired` / `sad` — выводятся из (mood, energy).
Грустный отдел → грустный питомец, как и просили.

## Развитие геймификации

- **Стрики**: дни подряд с закрытыми задачами / без просрочек (на команду и на
  сотрудника).
- **Квесты/челленджи**: недельные цели команды («закрыть N задач», «0 просрочек»)
  → кормят питомца и дают бонусный XP.
- **Уровни питомца**: командный XP растит level; новые «скины»/способности.
- **Отражение настроения**: цель — не «наказание», а наглядный командный сигнал
  благополучия (питомец грустит → менеджеру повод поговорить с командой).

## Интеграция

- **Вычисление**: периодический job (рядом с дайджест-джобами в
  `infrastructure/scheduler/runner.py`) пересчитывает mood/energy и применяет
  decay.
- **Чат**: показывать питомца в `/leaderboard` и в дайджесте; событийные
  реакции («Питомец загрустил — 3 просрочки на этой неделе»).
- **Mini-app** (`tgapp`): виджет питомца с анимацией состояния + прогресс уровня.
- **Связь с эмоциями**: mood читает агрегат `EmotionPortrait` по отделу (мягкая
  зависимость; работает и без эмоционального модуля).

## Этапы реализации

1. `TeamPetModel` + миграция + базовый расчёт mood по задачам (без эмоций).
2. Decay/кормление через существующие XP-события.
3. Виджет в `tgapp` + строка в `/leaderboard`/дайджесте.
4. Стрики и недельные челленджи.
5. Подключение эмоциональной компоненты (когда готов `emotional-portrait`).
