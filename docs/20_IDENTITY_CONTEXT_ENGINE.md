# Identity / Context Engine

`IdentityResolver` отделяет языковое понимание от привязки к реальному сотруднику. LLM
возвращает только текстовую ссылку на исполнителя, а backend разрешает её внутри команды.

Приоритет: Telegram `text_mention`, `@username`, автор сообщения в reply, `user_aliases`,
YouGile mapping, затем текстовая ссылка LLM. Результат содержит статус, источник,
confidence и список кандидатов. Для русских имён автоматически создаются базовые формы,
включая дательный падеж вроде `Денису`.

Reply хранится нормализованно в `chat_messages`: Telegram ID исходного сообщения,
локальный и Telegram ID автора, текст, thread ID. Поэтому `/task` в reply назначает
автора исходного сообщения, а короткий status update ищет связанную задачу, не
«последнюю задачу пользователя».
